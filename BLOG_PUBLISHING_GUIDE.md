# How to Publish Your Blog Post

Your blog post is ready in `BLOG_POST.md`. Here's how to publish it on different platforms for **+0.4 bonus points** in the hackathon.

---

## Option 1: Medium (Recommended - Best Reach)

### Step 1: Create/Login to Medium
- Go to https://medium.com/
- Sign up or log in

### Step 2: Start Writing
- Click "Write" button (top right)
- You'll see the Medium editor

### Step 3: Copy Content
1. Open `BLOG_POST.md` in a Markdown viewer or VS Code
2. Copy the entire content
3. Paste into Medium editor

### Step 4: Format
Medium will automatically convert most Markdown:
- Headers will work
- Code blocks will be formatted
- Links will be clickable

**Manual fixes needed:**
- Add the architecture diagram as an image:
  1. Convert `architecture-diagram.svg` to PNG (use `./generate-diagram-png.sh` or CloudConvert)
  2. Upload PNG in Medium editor where you want it
  3. Suggested location: After "Architecture Overview" section

### Step 5: Add Tags
Add these tags to increase visibility:
- Cloud Run
- Google Cloud
- AI Agents
- ADK
- Multi-Agent Systems
- Serverless
- FastAPI

### Step 6: Publish
1. Click "Publish" button
2. Add a subtitle (optional): "A technical deep-dive into building AI marketing automation with Google's Agent Development Kit"
3. Choose "Allow responses"
4. Click "Publish now"

### Step 7: Get URL for Hackathon
Copy the URL of your published post and add it to your Devpost submission.

---

## Option 2: Dev.to (Best for Developers)

### Step 1: Create/Login to Dev.to
- Go to https://dev.to/
- Sign up or log in with GitHub

### Step 2: Create Post
- Click "Create Post" (top right)
- You'll see a Markdown editor (perfect!)

### Step 3: Paste Content
1. Copy entire content from `BLOG_POST.md`
2. Paste directly into dev.to editor
3. **Dev.to supports Markdown natively** - no conversion needed!

### Step 4: Add Cover Image
1. Convert `architecture-diagram.svg` to PNG
2. Upload as cover image (recommended size: 1000x420px)

### Step 5: Add Tags
Dev.to allows up to 4 tags:
- `cloudrun`
- `googlecloud`
- `ai`
- `multiagent`

### Step 6: Publish
1. Click "Publish" button
2. Choose "Public" visibility
3. Confirm publish

### Step 7: Get URL for Hackathon
Copy the URL (format: `https://dev.to/username/title-xxx`) and add it to your Devpost submission.

---

## Option 3: Hashnode (Best for Tech Blogging)

### Step 1: Create Account
- Go to https://hashnode.com/
- Sign up with GitHub/Google

### Step 2: Create Your Blog
- Set up your blog subdomain (e.g., `yourname.hashnode.dev`)

### Step 3: Write Post
- Click "Write" or "New Post"
- Hashnode supports Markdown

### Step 4: Paste Content
1. Copy from `BLOG_POST.md`
2. Paste into Hashnode editor
3. Add cover image (architecture diagram PNG)

### Step 5: Add Tags
- Cloud Run
- Google Cloud Platform
- AI
- Multi-Agent Systems
- Serverless

### Step 6: Publish
1. Click "Publish"
2. Share on social media (optional)

---

## Option 4: Your Own Blog (If You Have One)

If you have a personal blog (WordPress, Ghost, Jekyll, Hugo, etc.):

1. Convert `BLOG_POST.md` to your blog's format
2. Upload architecture diagram image
3. Publish
4. Ensure it's **publicly accessible** (not behind paywall or auth)
5. Use the URL for hackathon submission
